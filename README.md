## **1.1 Subject of the First Three Texts**

Across all three authors (Lovecraft, RJ Martin, RJ Tolkien), the most frequent tokens and trigrams consistently include:

- **romeo**
- **juliet**
- **verona**
- **love**
- **fate**
- **ancient**
- **tale**

The trigrams reinforce this:

- `(',', 'romeo', 'juliet')`
- `('romeo', 'juliet', ',')`
- `('city', 'verona', ',')`
- `('land', 'verona', ',')`

### **Conclusion:**
All three texts are stylistically distinct retellings of **Romeo and Juliet set in Verona**, each in the voice of a different author:

- **Lovecraft:** cosmic, eldritch, fatalistic tone  
- **RJ Martin:** grounded, political, house‑driven drama  
- **RJ Tolkien:** mythic, fate‑woven, high‑fantasy tone  

---

## **1.2 Authorship Analysis of Text_4**

Text_4 differs sharply from the first three:

- No references to **Romeo**, **Juliet**, or **Verona**
- Heavy use of:
  - character names (`aldric`, `alysanne`, `torran`)
  - introspective verbs (`could`, `feel`)
  - physical descriptors (`cold`, `shadow`, `eye`)
  - dialogue punctuation (`''`, “ ”, ’)

The style is:

- grounded,
- gritty,
- character‑driven,
- political‑fantasy oriented.

### **Conclusion:**
Based on vocabulary, trigrams, and narrative tone, **Text_4 most closely resembles RJ Martin’s style**.  
It is highly likely that **RJ Martin is the author of Text_4**.

# 2. **Purpose of the Project**

The purpose of this project is to demonstrate how Natural Language Processing (NLP) techniques can extract structure, meaning, and stylistic patterns from unstructured text. By applying tokenization, stemming, lemmatization, named entity recognition, and trigram analysis, we can:

- identify thematic content,
- compare writing styles,
- quantify linguistic features,
- and infer authorship.

This project highlights how computational methods can support literary analysis, authorship attribution, and text classification.

---

# 3. **NLP Pipeline Overview**

The project uses a modular, step‑by‑step NLP pipeline:

1. **Load text**  
2. **Tokenize**  
   - original‑case tokens for NER  
   - lowercase tokens for frequency analysis  
3. **Remove stopwords**  
4. **Stem tokens**  
5. **Lemmatize tokens**  
6. **POS‑tag original tokens**  
7. **Run Named Entity Recognition**  
8. **Compute top 20 tokens**  
9. **Compute top 5 trigrams**  
10. **Aggregate results into a combined DataFrame**  
11. **Export to CSV**

This structure ensures clarity, reproducibility, and easy comparison across texts.

# 4. **Function Documentation**

Below is a breakdown of each function used in the project.

---

## **4.1 `open_read(path)`**
**Input:** file path  
**Output:** raw text string  
**Purpose:** Reads text files into memory.

---

## **4.2 `tokenize_text(txt_file)`**
**Input:** raw text  
**Output:** list of tokens (original case)  
**Purpose:** Used for POS tagging and NER.

---

## **4.3 `tokenize_lower(txt_file)`**
**Input:** raw text  
**Output:** list of lowercase tokens  
**Purpose:** Used for frequency analysis.

---

## **4.4 `remove_stop_words(tokenized_txt)`**
**Input:** token list  
**Output:** token list without stopwords  
**Purpose:** Removes common English words to highlight meaningful content.

---

## **4.5 `pos_tagging(tokens)`**
**Input:** token list  
**Output:** list of (token, POS tag) pairs  
**Purpose:** Required for NER.

---

## **4.6 `stem_words(filtered_txt)`**
**Input:** token list  
**Output:** list of stems  
**Purpose:** Assignment requirement; not used for final metrics.

---

## **4.7 `lemmatize_words(filtered_txt)`**
**Input:** token list  
**Output:** list of lemmas  
**Purpose:** Used for top token and trigram analysis.

---

## **4.8 `most_common(lemmatized_txt)`**
**Input:** list of lemmas  
**Output:** top 20 `(token, count)` pairs  
**Purpose:** Identifies key vocabulary.

---

## **4.9 `chunking(pos_tags)`**
**Input:** POS‑tagged tokens  
**Output:**  
- NER chunk tree  
- Named entity count  
**Purpose:** Extracts real‑world entities (people, places, organizations).

---

## **4.10 `most_common_n_grams(lemmatized_txt)`**
**Input:** list of lemmas  
**Output:** top 5 trigrams  
**Purpose:** Captures stylistic and thematic patterns.

# 5. **Limitations**

- **Punctuation appears in token lists**  
  Because punctuation is treated as tokens by NLTK and not removed.

- **NER model is basic**  
  NLTK’s built‑in NER is less accurate than modern transformer‑based models.

- **No semantic modeling**  
  The analysis is based on surface features, not deep meaning.

- **Functional design instead of class‑based**  
  Simpler for this assignment, but not ideal for large‑scale systems.
